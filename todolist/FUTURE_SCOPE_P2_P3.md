# WellFit – Future Scope P2/P3

Version: 1.0
Stand: 2026-05-01
Zweck: Spaetere Ausbau- und Visionsbereiche erhalten, aber von Alpha/Beta trennen

---

## Grundsatz

P2/P3 wird nicht geloescht.

Diese Themen bleiben Teil der WellFit-Vision, duerfen aber P0-Alpha und P1-Beta nicht blockieren.

---

## P2 – Ausbau nach stabiler Beta

P2 ist wertvoll, sobald das Kernprodukt stabil getestet wurde.

### P2-Themen

```txt
Voice Push-to-Talk
TTS-Ausgabe fuer Buddy
Avatar Voice Profiles
mehr Avatar-Typen
umfangreiche Ability-/Item-Profile
fortgeschrittene KI-Missionsempfehlungen
Backend Event API produktionsnah
Product UI Lokalisierung
familien-/altersabhaengige Varianten
Partner-/Museum-/Stadt-Missionsmodule
B2B-/Schul-/Tourismus-Pfade
```

### P2-Regel

```txt
P2 darf vorbereitet und dokumentiert werden.
P2 darf nicht P0/P1 blockieren.
P2 darf erst implementiert werden, wenn Kernfluss stabil ist.
```

---

## P3 – Vision / spaeter

P3 ist grosse Produktvision und strategischer Langfrist-Ausbau.

### P3-Themen

```txt
echte NFTs
echter WFT-Token
Blockchain-Integration
Marketplace
Trading
Staking
DAO
Presale in produktnahen Bereichen
DePIN / Compute-as-a-Service
komplexe Avatar-Vererbung
lebenslanger Avatar
Wake Word / Always Listening
freier Voice Chat
komplexe B2B-Whitelabel-Plattform
voller Partner-Marktplatz
```

### P3-Regel

```txt
P3 bleibt dokumentiert.
P3 wird nicht als aktueller Sprint behandelt.
P3 kommt erst nach belastbaren Alpha-/Beta-/Testdaten in aktive Planung.
```

---

## App-Store-/Compliance-Hinweis

Mobile App bleibt vorerst frei von:

```txt
Token
Presale
Trading
Staking
NFT-Marktplatz
Investment-Kommunikation
```

Solche Funktionen gehoeren spaeter, falls ueberhaupt, in getrennte Web-/Dashboard-/Compliance-gepruefte Bereiche.

---

## Warum P2/P3 erhalten bleiben

Diese Ideen sind strategisch wichtig fuer:

```txt
Investor Story
Produktvision
langfristige Differenzierung
B2B-/Partnerstrategie
Token-/Economy-Planung spaeter
```

Aber sie sind nicht noetig fuer:

```txt
erste Alpha
ersten Android-Test
ersten Buddy Game Loop
erste interne Punkte-/XP-Erfahrung
```

---

## Entscheidungsregel

Wenn eine neue Idee auftaucht:

```txt
Ist sie fuer Alpha noetig? -> P0
Macht sie Beta deutlich besser? -> P1
Ist sie Ausbau nach Beta? -> P2
Ist sie grosse Vision? -> P3
```
