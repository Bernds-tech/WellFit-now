## Verbindliche Regel

MissionDrafts sind nur Vorschlaege.

```txt
KI erzeugt Draft
Server/Policy prueft Draft
nur allowed/approved Drafts duerfen sichtbar oder nutzbar werden
```

## Geplante Collections

```txt
missionDrafts
missionDraftReviews
missionDraftApprovals
```

## Client-Grenze

```txt
Client darf MissionDrafts nicht schreiben.
Client darf MissionDraftReviews nicht schreiben.
Client darf MissionDraftApprovals nicht schreiben.
Client darf keine Punkte, XP, Completion oder Ledger in Drafts setzen.
```

## Emulator-Testziele

```txt
Client create missionDrafts -> PERMISSION_DENIED
Client update missionDrafts -> PERMISSION_DENIED
Client write missionDraftReviews -> PERMISSION_DENIED
Client write missionDraftApprovals -> PERMISSION_DENIED
Client read allowed Draft -> erlaubt
Client read raw draft status -> PERMISSION_DENIED
```
