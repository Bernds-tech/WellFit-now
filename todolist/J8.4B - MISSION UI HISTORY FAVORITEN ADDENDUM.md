## UI-Status

Vorhandene Missionsseiten muessen spaeter klar unterscheiden:

```txt
placeholder = Platzhalter
kiDraft = KI-Vorschlag
needsReview = In Pruefung
approved = Freigegeben
active = Aktiv
completed = Abgeschlossen
rejected = Nicht verfuegbar
```

## Wichtig

```txt
Platzhalter sind keine fertigen Missionen.
KI-Drafts sind keine freigegebenen Missionen.
Approved Missionen duerfen gestartet werden.
Completed darf erst nach Serverentscheidung erscheinen.
```

## History/Favoriten

Favoriten und History existieren als Seiten, sind aber keine Reward-Autoritaet.

```txt
Favoriten = gemerkte Missionen/Drafts
History = Verlauf/abgeschlossene oder gepruefte Ereignisse
```

AR-Buddy-Nebenmissionen koennen spaeter in History auftauchen als:

```txt
Side Quest Event
Evidence beantwortet
Reward geprueft
```

Aber nicht automatisch als:

```txt
Tagesmission abgeschlossen
Wochenmission abgeschlossen
Abenteuer abgeschlossen
Challenge abgeschlossen
Wettkampf abgeschlossen
```

## Sicherheitsgrenze

UI-Status, Karten, Standort, History und Favoriten duerfen niemals Punkte, XP, Reward, Completion oder Ledger autorisieren.

```txt
Standortdaten sind Kontext/Evidence, keine Reward-Autoritaet.
KI-Drafts sind Vorschlaege, keine freigegebenen Missionen.
Server/Policy entscheidet Completion, Rewards, Caps, Ledger und Wettkampf-Sieger.
```
