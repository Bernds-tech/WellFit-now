## Lokaler Setup-Ablauf

```bash
# Repository-Root oeffnen.
cd /path/to/WellFit-now

# Abhaengigkeiten nur installieren, wenn node_modules fehlt oder sichtbar veraltet ist.
npm install

# Lokale Env-Datei aus Vorlage erstellen und mit lokalen Werten fuellen.
cp .env.example .env.local

# Validierung vor Handoff/PR.
npm run lint
npx tsc --noEmit
npm run build

# Lokale Vorschau nur bei Bedarf starten.
npm run dev
```

PowerShell-Variante fuer Windows, falls `cp` nicht verfuegbar ist:

```powershell
copy .env.example .env.local
notepad .env.local
npm run build
```

## Sicherheitsregel

[!] `.env.local` darf nicht mit echten Werten ins Repository committed werden.
[!] `.env.example` bleibt eine Platzhalter-Vorlage; keine echten Firebase-Projektwerte oder Provider-Keys eintragen.
[!] Clientseitig sichtbare Firebase-Web-App-Variablen muessen mit `NEXT_PUBLIC_` beginnen.
[!] Echte Auth-/Firestore-Nutzung braucht konfigurierte Firebase-Werte in `.env.local` oder in der Deployment-Umgebung.
[!] CI/Codex darf oeffentliche Seiten ohne echte Firebase-Werte bauen, solange keine echte Auth-/Firestore-Nutzung im Build erzwungen wird.
[!] Serverseitige API-Keys duerfen niemals mit `NEXT_PUBLIC_` beginnen.
[!] Agenten duerfen fuer Setup-Dokumentation keine Runtime-Dateien, Produktlogik, Unity-Dateien, Secrets oder Deployment-Konfiguration aendern.
