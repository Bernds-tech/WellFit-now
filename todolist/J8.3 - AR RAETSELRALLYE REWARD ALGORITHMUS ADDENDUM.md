# WELLFIT – J8.3 AR Raetselrallye Reward Algorithmus Addendum

Stand: 2026-04-28
Status: Ergaenzung zu `J - NÄCHSTE EMPFOHLENE ARBEIT` und `G1 - INTERNE PUNKTEOEKONOMIE VOR BLOCKCHAIN`

## Zweck

Dieses Addendum dokumentiert die wichtige Produkt- und Architekturentscheidung fuer AR-Buddy-Fragen, Objekterkennung, Standort-Raetsel und Raetselrallyes.

## Verbindliche Entscheidung

AR-Buddy, Unity, Kamera-KI, Mobile UI und Buddy-KI duerfen keine Punktehoehe festlegen.

Sie duerfen nur erzeugen:

```txt
AR-Event
Objekt-/Kontext-Evidence
Fragevorschlag
Antwortauswahl
Buddy-Reaktion
```

Die Punktehoehe wird ausschliesslich serverseitig durch den bestehenden bzw. geplanten Punkte-/Reward-Algorithmus berechnet.

## Bezug zu G1

G1 bleibt verbindlich:

```txt
Punkte zuerst
Server-Ledger zuerst
Anti-Cheat zuerst
Beta-Test zuerst
Blockchain danach
```

Die geplante 25-Mrd.-WFT-Logik bleibt aktuell interne Punkte-/Economy-Simulation.

## Zielablauf fuer AR-Fragen

```txt
1. Kamera/KI erkennt Objekt oder Standortkontext
2. Buddy formuliert Aufgabe oder Frage
3. Nutzer waehlt Antwort
4. App erzeugt arQuestionAnswered Evidence
5. Backend prueft Antwort und Kontext
6. Question Memory prueft Wiederholung/Farming
7. Reward Policy berechnet erlaubte Punkte/XP
8. Server-Ledger schreibt Reward Event
9. Buddy zeigt Ergebnis emotional an
```

## Harte Nicht-Erlaubt-Regeln

```txt
Unity setzt keine Punkte
Kamera setzt keine Punkte
Buddy-KI setzt keine Punkte
Frontend setzt keine Punkte
Client entscheidet keine Mission Completion
Client entscheidet keine Rewardhoehe
```

## Beispiel Blattfrage

Der Buddy darf sagen:

```txt
Hey, schau mal, was ist das fuer ein Blatt?
```

Die UI darf Antworten anzeigen:

```txt
Eichenblatt
Ahornblatt
Birkenblatt
```

Nach Antwort darf die App nur Evidence senden.

Nicht erlaubt:

```txt
Buddy/Kamera/UI sagt sofort: +3 Punkte garantiert
```

Erlaubt nach Serverantwort:

```txt
Backend hat +3 Punkte freigegeben
Buddy freut sich
UI zeigt +3 Punkte gutgeschrieben
```

## Reward Policy muss beruecksichtigen

```txt
Aufgabentyp
Schwierigkeit
Antwort korrekt/falsch
Nutzerhistorie
Buddy-/User-Level optional
Question Novelty
Location Novelty
Wiederholungsstrafe
Daily User Cap
Mission Type Cap
Systemreserve
Economy Health Score
Risk Level
Evidence Quality
```

## Question Memory muss speichern

```txt
Welche Frage wurde gestellt?
Welches Objekt wurde erkannt?
An welchem Ort/Gebiet wurde gefragt?
Wie oft wurde sie gestellt?
Wie oft richtig/falsch beantwortet?
Wann darf sie wiederkommen?
Muss die Schwierigkeit steigen?
Gibt es Farming-Risiko?
```
