# WELLFIT – J8.4 Missionstypen und KI Mission Engine Addendum

Stand: 2026-04-28
Status: Ergaenzung zu `J - NÄCHSTE EMPFOHLENE ARBEIT`, `F - FIREBASE - REALTIME - MISSIONEN` und `J8.3`

## Zweck

Dieses Addendum dokumentiert die Abgrenzung zwischen vorhandenen Missionsseiten, Platzhalter-Missionen, KI-generierten MissionDrafts und AR-Buddy-Nebenmissionen.

## Verbindliche Klarstellung

Die Missionsseiten existieren bereits:

```txt
/missionen/tagesmissionen
/missionen/wochenmissionen
/missionen/abenteuer
/missionen/challenge
/missionen/wettkaempfe
/missionen/favoriten
/missionen/history
```

Aber:

```txt
Die Missionen darin sind aktuell Platzhalter / vorbereitete Container.
Fertige Missionen sollen spaeter durch KI-Buddy-/Mission-Engine, Server-Policy und kuratierte Inhalte entstehen.
```

## Missionstypen

```txt
daily = Tagesmission
weekly = Wochenmission
adventure = Abenteuer
challenge = Challenge
competition = Wettkampf
arSideQuest = AR-Buddy-Nebenmission
```

## AR-Buddy-Nebenmissionen

AR-Buddy-Nebenmissionen sind spontane kleine Zusatzaufgaben im Nahbereich.

Beispiel:

```txt
Was ist das fuer ein Blatt?
```

Sie zaehlen nicht automatisch als:

```txt
Tagesmission
Wochenmission
Abenteuer
Challenge
Wettkampf
```

Sie erzeugen zuerst nur Evidence.

## KI-Buddy-/Mission-Engine

Die KI darf:

```txt
Missionen draften
ortsnahe Rallyes vorschlagen
altersgerechte Aufgaben formulieren
Schwierigkeit vorschlagen
Mission Category vorschlagen
Antwortoptionen vorschlagen
Buddy-Dialog vorschlagen
```

Die KI darf nicht:

```txt
Mission final freigeben
Punktehoehe autorisieren
Reward-Ledger schreiben
Mission Completion autorisieren
Wettkampf-Sieger final bestimmen
Partner-Challenge ohne Serverfreigabe aktivieren
```

## Server-/Policy-Aufgaben

Server/Policy entscheidet:

```txt
Kategorie
Freigabe
Sichtbarkeit
Reward Policy
Caps
Anti-Cheat
Completion
Ledger
History-Eintrag
Favoriten-Verhalten
```

## Sicherheitsgrenze

Keine Mission, egal aus welcher Kategorie, darf clientseitig Punkte, XP, Completion oder Ledger schreiben.

```txt
KI erzeugt Draft
Server prueft und erlaubt
Nutzer erzeugt Evidence
Reward Policy berechnet
Server Ledger schreibt
```
