B. AKTUELLE CODE-ANALYSE REALTIME / DASHBOARD
============================================================

TASK-ID: WF-DASH-004
STATUS: [~] In Arbeit
PRIORITÄT: nächster technischer Schritt
ZIEL:
Dashboard soll live mit Firebase synchronisieren, ohne dass User neu laden müssen.
AKTUELLER CODE-STAND:
[x] Dashboard nutzt Firebase Auth mit onAuthStateChanged.
[x] Dashboard nutzt lokalen Cache, dadurch sind Daten schneller sichtbar.
[x] Dashboard liest aktuell Firestore über getDoc.
[x] Dashboard schreibt Fortschritt über setDoc mit merge.
[x] Punkte, Schritte, Level und Avatarwerte bleiben nach Reload erhalten.
[x] lastLoginAt / updatedAt werden beim Dashboard-Laden aktualisiert.
[ ] Dashboard nutzt noch kein onSnapshot für echte Live-Synchronisation.
[ ] Änderungen aus anderen Tabs/Geräten erscheinen aktuell nicht automatisch live.
[ ] Cache + Realtime müssen sauber kombiniert werden.
EINSCHÄTZUNG:
Ein großer Teil des gewünschten Verhaltens ist bereits erledigt: schnelleres Laden, Firebase-Speicherung, Cache, Reload-Stabilität.
Was noch fehlt, ist echte Firestore-Realtime-Synchronisation mit onSnapshot.
UMSETZUNG NÄCHSTER SCHRITT:
[ ] getDoc im Dashboard durch onSnapshot ergänzen/ersetzen.
[ ] Cache sofort anzeigen.
[ ] onSnapshot danach als Live-Wahrheit verwenden.
[ ] Cache bei jedem Snapshot aktualisieren.
[ ] Loading-State sauber beenden, sobald erster Snapshot kommt.
[ ] Listener beim Verlassen der Seite sauber unsubscriben.
[ ] Schreiboperationen weiter mit setDoc/updateDoc merge halten.
AKZEPTANZKRITERIEN:
[ ] Änderung in Firestore erscheint im Dashboard ohne Reload.
[ ] Änderung in zweitem Browser/Tab erscheint live.
[ ] Cache zeigt Daten sofort, Snapshot korrigiert danach.
[ ] Keine Endlos-Loading-Anzeige.
[ ] Keine doppelten Listener.
[ ] Kein Datenverlust durch lokale Zwischenstände.
